Опис можливого «проєкту»

  1. Яку проблему вирішуємо? (1 пропозиція)

    Закріплення навичок рахунку дітей дошкільного віку.

  2. Як вирішуємо основну проблему? (послідовність дій)

    1. Визначити основні компоненти застосунку. Наприклад, кожне виконане завдання буде новим кубиком. Для того, щоб перейти на наступний рівень, потрібно скласти вежу з усіх кубиків. Тобто частину екрану буде займати ця вежа, що будується. Далі кнопка "Готово" для підтвердження результату дії. Також потрібна іконка на відтворення звуку, що супроводжувати завдання голосом.
    2. Визначити, скільки екранів потрібно для найпростішої реалізації застосунку. Наприклад, для початку достатньо два: один - карта завдань, другий - для відображення завдань.
    3. Розробити інтерфейс застосунку у графічному редакторі.
    4. Починаємо розробку, пишемо код.
    5. Шукаємо баги.
    6. Віддаємо в реліз.

  3. Яка структура проєкту? (список сторінок / екранів проєкту, наприклад)

    • Перший екран - для відображення карти завдань. Це може бути карта з позначеними кружечками пунктами з кількістю зірочок, отриманих за виконаний рівень.
    • Другий екран - для відображення самого завдання з вежею з кількістю вірно пройдених завдань рівня, самим завданням (типу 3 + 2) у вигляді якихось предметів або цифрами, варіантами відповідей, кнопкою "Готово" для відправки завдання, місцем для отриманих зірочок.

  4. Як буде виглядати інтерфейс проєкту? (приблизний опис)

  5. Опишіть функціонал проєкту (на прототипі вказати цифрами основні функційні елементи. Скласти нумерований список, у якому кожен пункт є описом функційного елементу, зазначеного цифрою на прототипі, коротко і приблизно)

    1. Екран “Завдання”.
    2. Вежа успіху: кількість кубиків по кількості виконаних завдань.
    3. Кнопка “Готово” для відправки завдання на перевірку.
    4. Зірочки, що показують кількість співвідношення кількості правильно виконаних завдань.
    5. Текст завдання.
    6. Варіанти відповідей.
    7. Екран “Карта завдань”.
    8. Кружечки-рівні.
    9. Зірочки до кожного рівня.
    10. Транспорт, який показує, на якій позиції знаходиться студент. Транспорт можна обрати.

  6. Створення MVP (опишіть ту функційність проєкту, при реалізації якої він уже буде давати результат, але витрати на його реалізацію будуть мінімальними)

    Для початку, головне - наявність першого екрану, де діти можуть вже розвивати свої навички рахунку і бачити, як рухаються завданнями.

  7. Опишіть послідовність реалізації проєкту (список функцій, які будуть втілені після реалізації MVP)

    Потім потрібно додати другий екран з картою, де можна обирати транспорт, яким можна переміщатися по карті. Також додатково можна реалізувати інші екрани: типу онлайн-змагання, мої успіхи, намалюй собі зірочку, ускладнені завдання.
